## Overview

The `gw3-branch` skill provides coordinated branch management across the GW3 multi-repo workspace. It operates on three tiers: the prplos repository (project tier), feed repositories (feed tier, under `aei/<feed>/`), and module repositories (module tier, under `aei/src/<pkg>/`).

## Architecture

### Repository Hierarchy

```
prplos/                          (main repo)
├── profiles/*.yml               (feed definitions with revision tracking)
├── aei/                         (gitignored — local development)
│   ├── <feed>/                  (Feed tier: feed repos)
│   │   ├── .git/
│   │   └── <pkg>/Makefile       (PKG_SOURCE_VERSION tracking)
│   └── src/
│       └── <pkg>/               (Module tier: module repos)
│           └── .git/
└── feeds.conf                   (generated by gen_config.py, modified by src_local.sh)
```

### Branch Types

| Type | Pattern | prplos branch | Feed branch | Source branch | Feed revision in YAML |
|------|---------|--------------|-------------|--------------|----------------------|
| Mainline | `r3_mainline` | `r3_mainline` | `main` | N/A (no checkouts) | Commit-pinned (^sha1) |
| Feature | `feature/<name>` | `feature/<name>` | `feature/<name>` | `feature/<name>` | Branch-tracked (;branch) |
| Hotfix | `hotfix/<name>` | `hotfix/<name>` | `hotfix/<name>` | `hotfix/<name>` | Branch-tracked (;branch) |
| Release | `release/<ver>` | `release/<ver>` | N/A (commit-pinned) | N/A | Commit-pinned (^sha1) |

### Feed Classification

- **GW3-owned feeds**: Defined in profile YAMLs (`feed_aei`, `feed_amx`, `feed_prplos`, `feed_prplmesh`, `feed_lcm`, `feed_qca`). Subject to branch management.
- **Third-party feeds**: From `feeds.conf.default` (`packages`, `luci`, `routing`, `telephony`). Not managed by this skill.

## Subcommand Specifications

### SC-01: Status (no args)

**Purpose**: Show workspace state with adaptive verbosity.

**Logic**:
1. Get prplos branch and classify type (main/feature/release/hotfix/unknown)
2. Discover all git repos under `aei/<feed>/` and `aei/src/<pkg>/`
3. For each repo: check branch name, dirty status
4. On feature/hotfix: warn if any repo branch doesn't match prplos branch
5. On main/release: check profile YAML feed revisions are commit-pinned; check `PKG_SOURCE_VERSION` in Makefiles
6. On main/release with aei/ checkouts: warn about cleanup
7. Output brief if no warnings and not verbose; detailed otherwise

**Output formats**:
- Brief: `prplos @ <branch> | N feed(s), M module(s) | all clean`
- Detailed: Multi-line with feeds, sources, and warnings sections

### SC-02: Pull

**Purpose**: `git pull` on all repos in the workspace.

**Logic**:
1. `git pull` on prplos
2. `git pull` on each `aei/<feed>/` with `.git`
3. `git pull` on each `aei/src/<pkg>/` with `.git`
4. On failure: warn and continue (don't abort)
5. Print status report

### SC-03: Commit

**Purpose**: Coordinated commit across all dirty repos.

**Feature/hotfix branch logic**:
1. Iterate module repos → feed repos → prplos (inner-to-outer order)
2. For each dirty repo: `git add -A`, commit with generated message
3. Message format: "Update `<name>` for `<branch>`" with `--stat` in body
4. Print push hint

**Main/release branch logic**:
1. Check feed revisions in profile YAMLs — all must be commit-pinned
2. Check `PKG_SOURCE_VERSION` in package Makefiles — must be 40-char hex
3. If any unpinned: print error list, suggest `src_local.sh --pin`, exit 1
4. If all pinned: commit only prplos

### SC-04: Push

**Purpose**: Push all repos with local commits ahead of remote.

**Logic**:
1. For each repo (prplos, feeds, modules):
   - Check `git rev-list @{u}..HEAD --count`
   - If > 0: `git push origin <branch>`
2. Report summary of pushed repos

### SC-05: Feature/<name>

**Purpose**: Create, switch to, or extend a feature branch.

**Additive behavior**: Running repeatedly with the same branch but different feeds/modules extends the workspace without disturbing existing checkouts.

**Branch exists (switch + extend)**:
1. Switch prplos if not already on branch
2. For specified feeds: clone/switch via `src_local.sh` (skip if already src-linked)
3. For `--deep` feeds: `src_local.sh --deep`
4. For `--source` packages: `src_local.sh --source`
5. If no args and no src-links: auto-detect tracked feeds from profiles
6. Ensure all existing `aei/src/` repos are on the correct branch

**Branch does NOT exist (create + setup)**:
1. If no feeds specified: show interactive menu of GW3-owned feeds
2. Create `feature/<name>` branch in prplos
3. Update profile YAMLs: change selected feed revisions to `feature/<name>`
4. Set up feeds via `src_local.sh`
5. Optionally set up module repos
6. Print status

### SC-06: Hotfix/<name>

**Purpose**: Create or switch hotfix branch.

**Requires**: `--from release/<ver>` when creating.

**Branch exists**: Delegates to feature switch logic.

**Branch does NOT exist**:
1. Checkout the release branch
2. Create `hotfix/<name>` from it
3. Update profile YAMLs for selected feeds
4. Set up workspace via `src_local.sh`

### SC-07: Release/<name>

**Purpose**: Create or switch release branch.

**Prerequisites**: All revisions must be pinned:
- Feed revisions in profile YAMLs (40-char SHA)
- `PKG_SOURCE_VERSION` in Makefiles (40-char hex)

**Logic**:
1. Verify no uncommitted changes
2. Verify all revisions pinned (both tiers)
3. Switch or create branch in prplos only

### SC-08: Main

**Purpose**: Switch to mainline, cleaning up workspace.

**Logic**:
1. Warn about uncommitted changes
2. Run `src_local.sh --clean` to restore feeds.conf
3. Switch prplos to `r3_mainline`
4. Run `feeds update -a && feeds install -a`
5. Print status

### SC-09: Sync

**Purpose**: Switch all feed and module repos to match the current prplos branch.

**Precondition**: prplos must be on a feature or hotfix branch. Errors on main/release (those require pinning, not branch sync).

**Logic**:
1. Get prplos branch and classify type
2. Reject if main or release — `die` with message
3. For each feed repo (`discover_feed_repos`):
   - Skip if not GW3-owned (`is_owned_repo`)
   - Skip if already on target branch
   - If dirty (`repo_is_dirty`): warn and skip, increment skipped counter
   - `git fetch origin`, then checkout target branch (local → remote tracking → new local)
   - Increment switched counter
4. For each module repo (`discover_source_repos`):
   - Same skip/dirty/checkout logic as feeds
5. Print summary: "Synced: N repo(s) switched, M skipped"
6. If any skipped: warn "Commit or stash dirty repos, then re-run sync"

**Safety**:
- Never discards uncommitted changes (dirty repos are skipped)
- Non-interactive — no prompts, no destructive operations
- Read-only with respect to working tree content (only changes branch pointer)

### SC-10: PR

**Purpose**: Automate the bottom-up multi-repo PR workflow (modules → feeds → prplos).

**Precondition**: prplos must be on a `feature/*` branch. `bkt` CLI must be authenticated.

**Options**:
- `--dry-run` — Show what would happen without rebase, push, PR creation, or pin updates
- `--reviewer <username>` — Override default Bitbucket reviewers (repeatable)

**Logic**:
1. **Validation**: Verify feature branch, `bkt` availability and auth
2. **Feed order**: Read from `aei/.pr-config.yaml` or use default (alphabetical, `feed_prplmesh` second-to-last, `feed_aei` last). Filter to localized feeds only
3. **Module processing** (per feed): For each module in `aei/src/` belonging to this feed:
   - Ensure clean (`repo_is_dirty` → exit 1)
   - Fetch + rebase onto `origin/main`; abort on conflict (exit 1)
   - Push with `--force-with-lease`
   - Detect state via divergence count (`rev-list --count origin/main..HEAD`):
     - `0` (merged): Pin `PKG_SOURCE_VERSION` in feed Makefile, set hashes to `skip`, auto-commit in feed, remove `aei/src/<module>/` and state entry
     - `>0` + open PR: Report PR ID, mark pending
     - `>0` + no PR: Create PR via `bkt pr create --close-source`
4. **Feed processing**: If all modules pinned, process the feed itself:
   - Same rebase/push/detect flow
   - Merged: Pin revision in profile YAML via `update_profile_revision()`, auto-commit in prplos
   - If any module PR pending: feed is "blocked"
5. **Project processing**: If all feeds pinned, process prplos:
   - Rebase onto `origin/r3_mainline`, push
   - Create PR with aggregate description (all feed/module changes)
   - If any feed PR pending: prplos is "blocked"
6. **Summary**: Print status table showing every repo's state and next steps

**Safety**:
- Exit 1 on any uncommitted changes or rebase conflicts
- `--force-with-lease` for all post-rebase pushes
- Pin to `origin/main` HEAD (handles squash merges)
- Auto-commit all pin updates with descriptive messages
- Idempotent — safe to re-run as PRs merge

**Implementation**: Defined in `gw3_branch_pr.sh`, sourced by `gw3_branch.sh`.

## Discovery Functions

### Profile YAML Parsing

Uses Python with PyYAML to parse all `profiles/*.yml` files. For each file, extracts feed entries with `name`, `uri`, `revision`, and maps them by feed name (last definition wins for duplicate names across profiles).

**Revision classification**:
- Commit-pinned: 40-char hex SHA-1, or `tag@40-char-sha1`
- Branch-tracked: anything else (branch name)

### Repository Discovery

- Feed repos: directories under `aei/` (excluding `aei/src/`) with `.git/`
- Module repos: directories under `aei/src/` with `.git/`

## Safety Rules

1. Always warn about uncommitted changes before branch switches
2. Never force-checkout or discard changes
3. Enforce pinning on main/release before commit
4. Pull failures don't abort — warn and continue
5. Push failures don't abort — warn and continue
6. Feature extension is additive — never removes existing src-links

## Dependencies

| Dependency | Purpose |
|-----------|---------|
| `src_local.sh` | Feed src-link setup, module checkout, cleanup, pinning |
